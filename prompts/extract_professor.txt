以下是你需要执行的流程，请谨慎遵守：

1. 输入内容
	- 我会提供目标学校名称、所属学院/系/部门名称，以及一段网页内容（已转换为 Markdown）。
	- 你的任务是在网页中查找并确认属于该学校指定部门的教授姓名。

2. 身份判定
	- 先确认网页描述的学校与部门是否与输入一致，可依据学校全称、缩写、地理位置、学院/系名称等信息。
	- 仅当你高度确信网页中人员隶属目标学校且服务于指定部门时，才可将其视为合格教授。
	- 无法确认身份或部门隶属时，宁可不返回该姓名。

3. 教授识别规则
	- 只提取教授及同等级别教职人员的姓名（如“教授”“副教授”“讲席教授”等明确称谓）。
	- 若网页包含其他人员（学生、助理、其他学院教师等），全部忽略。
	- 对于含糊描述（例如“我们学院的老师”）且无法确认具体姓名的，不输出。
    - 必须包含完整名字，如果只有姓没有名（例如王教授，李某），不输出，你需要判断教授是否真的叫李某，还是因为隐私原因不透漏

4. 输出格式
	- 以 JSON 列表返回，每个元素是一个字符串，对应一位教授的姓名。
	- 名称使用网页中的中文或英文原文，不做改写。
	- 如果没有任何可确认的教授，输出 `[]`。

5. 示例
	- 输入：
	  ```
	  目标学校：华东理工大学
	  目标部门：化学与分子工程学院
	  网页内容：...教授列表包含张三、李四（均为该院教授），另提到王五为材料学院教授...
	  ```
	- 输出：
	  ```json
	  [
	    "张三",
	    "李四"
	  ]
	  ```

6. 其他要求
	- 保守判断，宁缺毋滥。
	- 不要复述输入内容，也不要输出除教授姓名外的任何文本。
